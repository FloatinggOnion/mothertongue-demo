Create src/proxy.ts (Next.js 16 standard name, replaces old middleware.ts) to protect /drill and /history routes with a lightweight session cookie check. This is an optimistic guard — full validation happens in each page.

File contents:
```typescript
// src/proxy.ts
// Next.js 16 proxy — renamed from middleware.ts
// Source: https://nextjs.org/docs/app/getting-started/proxy + https://better-auth.com/docs/integrations/next
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";
import { getSessionCookie } from "better-auth/cookies";

export function proxy(request: NextRequest) {
  // Check for session cookie (optimistic check — not a security gate)
  const session = getSessionCookie(request);

  // If no session, redirect to login
  if (!session) {
    return NextResponse.redirect(new URL("/login", request.url));
  }

  // Session exists, allow through to page (page will do full validation)
  return NextResponse.next();
}

// Matcher: Apply proxy to protected routes only
export const config = {
  matcher: ["/drill/:path*", "/history"],
};
```

This proxy:
- Runs on every request to /drill/* and /history
- Checks for the session cookie (lightweight, no DB call)
- Redirects to /login if no cookie (prevents unauthenticated access)
- Allows authenticated requests through (page does full session validation)

CRITICAL: This is an optimistic guard, not a security gate. The actual page must call `auth.api.getSession()` to validate the token.
