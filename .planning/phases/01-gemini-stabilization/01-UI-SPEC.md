# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase UI Scope

This phase is a backend-hardening phase. The only new UI is:

1. **FeedbackCard error state** — replaces score/feedback content when evaluation fails; includes a "Retry evaluation" button (INFRA-01)
2. **Retry progress indicator** — inline text shown during exponential backoff: "Retrying... (attempt X of 3)" (INFRA-02)

All other UI is pre-existing and must not change. The design contract below covers both new elements and codifies the existing visual language so future phases extend it consistently.

- Primary CTA buttons (gradient `from-emerald-500 to-teal-500`)

**Borders:** `border-white/10` for all card borders. `border-white/20` on hover states.

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|

## Component Contract

### FeedbackCard Error State

The existing `FeedbackCard` component receives `evaluation: Evaluation` (always valid). Phase 1 introduces a discriminated union to support the error state.

**New prop signature (discriminated union — Pattern B from RESEARCH.md):**

```typescript
type FeedbackCardProps =
  | {
      state: 'success';
      evaluation: Evaluation;
      metrics: ConversationMetrics;
      onClose: () => void;
      onTryAgain: () => void;
    }
  | {
      state: 'error';
      errorMessage: string;
      onRetry: () => void;
      onClose: () => void;
    };
```

**Error state layout** (within the existing modal shell — same `fixed inset-0`, same `bg-gradient-to-br from-slate-900` card):

- Hide: score breakdown section, feedback panels (strength/improvement/corrected), metrics row
- Show: amber error panel with "Couldn't get your feedback" heading + body copy
- Show: two action buttons — "Back to Scenarios" (secondary, `bg-white/10`) and "Retry evaluation" (primary, amber gradient `from-amber-500 to-orange-500`)
- Keep: modal backdrop, card border, padding

**Error panel structure:**
```
[amber panel: bg-amber-500/10 border border-amber-500/30 rounded-xl p-4]
  [text-amber-400 font-medium text-sm] "Couldn't get your feedback"
  [text-white text-sm mt-1] "{evaluation error body copy}"
```

### Retry Progress Indicator

Location: Inside the loading state of the drill page — below the existing "Evaluating..." or loading spinner, centered.

**Visibility rule:** Only visible when a retry is actively in progress (attempt 2 or 3 of 3). On attempt 1 (first try), show normal loading state only. On retry, append the indicator below.

**Styling:**
```
[text-sm text-slate-400 text-center mt-2]
"Retrying... (attempt {n} of 3)"
```

**Implementation note:** The drill page uses frontend timer inference (see RESEARCH.md Pattern 1 — Simplest correct design). Show "Retrying..." after 3s elapsed since request sent; update attempt count based on cumulative elapsed time against backoff windows (2–4s, 8–16s). No backend callback needed.

## Interaction States

### FeedbackCard — All States

| State | Trigger | Visual |
|-------|---------|--------|
| success | evaluation completes normally | Full score breakdown, feedback panels, emerald/teal CTA |
| error | evaluateConversation() throws | Amber error panel, retry + back buttons, scores hidden |
| loading (pre-existing) | evaluation in progress | Not rendered — FeedbackCard appears only after result/error |

### Retry Indicator — States

| State | Trigger | Visual |
|-------|---------|--------|
| hidden | request < 3s elapsed | Not rendered |
| attempt 2 | request 3–12s elapsed | "Retrying... (attempt 2 of 3)" in slate-400 |
| attempt 3 | request 12s+ elapsed | "Retrying... (attempt 3 of 3)" in slate-400 |
| exhausted | all retries failed | Indicator disappears; FeedbackCard error state shown |
